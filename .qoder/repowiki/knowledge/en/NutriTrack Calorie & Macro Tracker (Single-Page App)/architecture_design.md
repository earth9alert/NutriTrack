Monolithic single-page application delivered by one `index.html` with no build step. The file is split into three logical layers: (1) `<head>` — Tailwind CSS loaded from CDN with an inline custom `brand` color palette plus hand-written CSS for the SVG calorie-ring (`stroke-dashoffset` animation), macro bars, and food-item slide-in; (2) `<body>` — static HTML sections for the dashboard (calorie ring + macro bars), preset quick-add grid, manual food-entry form, and four meal-grouped diary lists; (3) `<script>` — imperative vanilla-JS state machine centered on a single `foodLog` array persisted to `localStorage('nutritrack_log')`. Core functions follow a uniform add/delete → `save()` → `renderAll()` pipeline, where `renderAll()` recomputes totals via `Array.reduce` against a fixed `GOALS` constant and updates every DOM node by `id`. Preset foods are a hard-coded `PRESETS` array rendered into buttons whose `onclick` handlers call `addPreset(index)` which reads the selected meal from `#presetMealSelect`. There is no routing, component framework, or external JS dependency beyond Tailwind's runtime CDN.
